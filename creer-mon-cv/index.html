Créer mon CV structuré

Un script Python qui fabrique un PDF nativement balisé pour les ATS, à partir d'un simple fichier texte. Pas de design, pas de fantaisie : du contenu propre, lisible à 100% par les machines.

Pour qui ?

Plutôt si tu pars de zéro

Si tu as déjà un CV graphique qui te plaît, ce n'est probablement pas pour toi : utilise plutôt "Blinder mon CV". En revanche, si tu n'as pas de CV ou si tu veux quelque chose de chirurgicalement parsable (tech, ingé, fonctions support, scientifique, juridique…), c'est l'approche la plus solide.

L'idée : tu décris ton CV dans un fichier texte (YAML, très lisible), tu lances une commande, tu obtiens un PDF. Le PDF généré a une langue déclarée, des métadonnées propres, une structure linéaire, et passe le test d'extraction de texte sans rien perdre.

Installation

3 minutes, montre en main

Il te faut Python 3.9+ installé. Si tu ne sais pas si tu l'as, ouvre un terminal et tape python3 --version. Sur Mac et Linux c'est en général déjà là, sur Windows installe-le depuis python.org. 
# 1. Télécharger les fichiers (voir liste plus bas) dans un dossier
# 2. Ouvrir un terminal dans ce dossier
# 3. Installer les dépendances
pip install -r requirements.txt

# 4. Générer un CV à partir de l'exemple fourni
python creer_cv.py exemple_cv.yaml -o mon_cv.pdf

Le PDF mon_cv.pdf est créé dans le dossier courant. Ouvre-le pour vérifier le rendu.

Utilisation

Édite le YAML, relance la commande

Le fichier exemple_cv.yaml est ton point de départ. Ouvre-le avec n'importe quel éditeur de texte (TextEdit, Notepad, VS Code, ce que tu veux) et modifie tes infos. La syntaxe est minimaliste :

nom: Camille Dupont
email: camille.dupont@example.com
ville: Lyon, France

profil: >
  Chargée de communication digitale avec 3 ans d'expérience…

experiences:
  - poste: Chargée de communication
    entreprise: Association Lumières
    debut: 09/2022
    fin: 06/2024
    missions:
      - Pilotage de la stratégie digitale
      - Création de contenu graphique
      - Gestion des campagnes Meta Ads

Tu modifies, tu sauvegardes, tu relances la commande, le PDF se régénère. Pas de logiciel à apprendre, pas d'interface à comprendre.

Pour aller plus loin

Conformité PDF/UA stricte

Le script utilise reportlab, qui pose les bases du balisage (langue déclarée, métadonnées, structure linéaire propre). Pour 95% des ATS c'est largement suffisant.

Pour une conformité PDF/UA-1 stricte avec validation (utile surtout pour les lecteurs d'écran, pas vraiment pour les ATS), la voie est WeasyPrint : pip install weasyprint puis génération depuis du HTML sémantique. C'est dans EXPLICATIONS.md à la racine du projet.
